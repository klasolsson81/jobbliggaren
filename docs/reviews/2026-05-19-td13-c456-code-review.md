## Major

Inga. Specifikt verifierat icke-brott på de högrisk-punkterna:

- **Clean Arch (§2.1):** Domain orört — `ResumeVersion.Content` är
  `public ... { get; private set; }`, inga EF-/krypto-attribut. Port
  `IFieldEncryptionBackfiller` + `BackfillFieldEncryptionJob` +
  `LegacyFieldCounts` + `FieldEncryptionBackfillRun` ligger i Application,
  endast .NET BCL + `IDateTimeProvider`/`ISystemEventAuditor`/`ILogger`.
  Impl `FieldEncryptionBackfiller` i Infrastructure (EF/Npgsql isolerat dit).
  Paritet med `IAccountHardDeleter`-precedensen — porten exponerar primitiv
  `Guid`, impl wrappar `JobSeekerId`. Inget EF-/Npgsql-läckage in i Application.
- **DDD (§2.2):** #1c läcker ingen persistens in i aggregatet — `Content`
  förblir domän-property, `builder.Ignore(rv => rv.Content)`, shadow
  (`ContentEnc`/`ContentLegacyJson`) är Infrastructure-only shadow-properties.
  Backfiller force-Modified (`Entry(...).Property(name).IsModified = true`)
  muterar inte domäntillstånd — sätter bara EF change-tracking-flagga.
  `ResolveOwner` navigerar barn→spårad parent via FK, ingen aggregat-genväg.
- **CQRS/§3.5:** Inget sync-over-async. Inga `.Result`/`.Wait()`.
  `CancellationToken` propagerad genom hela kedjan (job→port→`SqlQueryRaw`/
  `ToListAsync`/`SaveChangesAsync`). `ConfigureAwait(false)` konsekvent i
  Infrastructure-porten. Orchestrator-loop bounded med `MaxBatchIterations`
  non-convergence-vakt + `OperationCanceledException` re-throw (sväljs ej).
- **§5.4:** Form B raw SQL är antingen parameterlös konstant-predikat
  (`GetOwnersWithLegacyFieldsAsync` Form B / `CountRemainingLegacyAsync`)
  eller parameteriserad (`ScalarIdsAsync` med `{0}`-bindning av `jobSeekerId`).
  Ingen string-concatenation av användardata. Per-owner fresh DI-scope via
  `IServiceScopeFactory` → ingen cross-user-DEK-ackumulering (§5.1).
  DEK zero:as direkt efter prefetch (`CryptographicOperations.ZeroMemory`).
  Ingen klartext-PII-loggning — `LoggerMessage` loggar bara `JobSeekerId`/
  räknare/iterationer.
- **§5.1:** `IAppDbContext`/`AppDbContext` direkt, ingen Repository-abstraktion.
  Ingen generisk `*Service`-suffix (`FieldEncryptionBackfiller`,
  `BackfillFieldEncryptionJob`, `AccountHardDeleter` namnger vad de gör).
  Ingen `DateTime.Now/UtcNow` — `IDateTimeProvider clock` injicerat i jobbet.
- **§3.6:** LINQ via `IgnoreQueryFilters().AsNoTracking()` på read/count;
  tracked load endast där write-interceptorn kräver spårad parent (motiverat
  i kommentar). Backfill bounded (`batchSize`/`OwnerBatchSize`/per-kolumn
  legacy-id-set).
- **§7 / testkvalitet:** Testcontainers-baserat (`WorkerTestFixture`,
  riktig Postgres), inte InMemory. Specifikationstest mot kontrakts-ytan
  (round-trip, on-disk-ciphertext, idempotens byte-identitet, fitness=0,
  cross-user, fail-closed, system-scope-passthrough) — inte rad-för-rad
  impl-spegling. Arch-test utökad (markör-krav + system-scope-tripwire +
  #1c-modell-invariant). Probe-deletion: rationale dokumenterad i
  `ResumeContentEncryptionTests.cs` XML-doc §"C4.0-proben subsumerades"
  (premissen "prod-modellen applicerar JSON-VC" föll med `builder.Ignore`;
  read-ordnings-invarianten utövas nu av scenario 1 mot verklig prod-
  interceptor) + ADR 0049 Mekanik-not 6 + git-historik. Legitim subsumtion,
  inte täckningsförlust.
- **feedback_di_with_handlers_same_commit:** Port-impl + job + worker-wrapper
  + RecurringJob + båda DI-registreringar är i samma uncommittade ändring.
  Ingen broken split-state.
- **DRY/SPOT:** `EncryptedFieldRegistry.ContentJsonOptions` är enda
  produktions-hemvist; `ResumeVersionConfiguration` JSON-VC + lokal
  `JsonSerializerOptions` borttagen; arch-test
  `ResumeVersionConfiguration_Honors_Number1c_Invariants` spärrar regression.

## Minor

1. **Sentinel `v1:`-litteral spridd över tre hemvister — SPOT-svaghet**
   Fil: `src/JobbPilot.Infrastructure/Security/FieldEncryptionBackfiller.cs:61`
   (`SentinelLikePattern = "v1:%"`) + inline raw-SQL `NOT LIKE 'v1:%'`
   (rader 131, 141, 151, 161, 269) + `KmsEnvelopeEncryptor.cs:23`
   (`CurrentVersionPrefix = "v1:"`, `private const`).
   Observation: den kanoniska sentinel-auktoriteten (`CurrentVersionPrefix`)
   är `private` i `KmsEnvelopeEncryptor` och exponeras inte på
   `IFieldEncryptor`-porten, så backfillern *kan* inte referera den utan en
   port-yt-utökning. CLAUDE.md §5.1 (magic strings) i bokstavlig mening träffas
   inte (det är en `const` + dokumenterad SQL-översättnings-approximation med
   uttrycklig motivering rad 53–60), men sanningen finns nu på 3 ställen utan
   kompilator-länk dem emellan. Risk realiseras först vid `v2:`-format (Fas 2+,
   dependency finns ej idag).
   §9.6-bedömning: **TD, inte in-block.** Detta hör till format-versionering
   (Fas 2 — `v2:` finns inte, ingen port-yta att exponera prefixet på idag).
   Att lyfta port-kontraktet `IFieldEncryptor` nu vore scope-expansion utan
   nuvarande-fas-dependency. Rekommendation: lyft TD "Sentinel-prefix SPOT —
   exponera CurrentVersionPrefix på IFieldEncryptor när v2-format införs"
   (Severity Minor, Fas 2). Inte in-block-fixbar utan port-ändring → korrekt
   TD-kandidat per §9.6 kriterium 1.

2. **`_ = startedAt;` discard utan förklaring**
   Fil: `src/JobbPilot.Application/Security/Jobs/BackfillFieldEncryption/BackfillFieldEncryptionJob.cs:51,113`
   `startedAt` sätts (`clock.UtcNow`) men konsumeras aldrig — endast
   `_ = startedAt;` för att undvika unused-warning. Antingen ta bort variabeln
   helt, eller inkludera den i audit-payloaden (`FieldEncryptionBackfillRun`
   har `OccurredAt` men ingen `StartedAt` — `JobAdsSynced` har både `StartedAt`
   och `CompletedAt` som precedens). Dead code i nuvarande form.
   §9.6: **in-block-fixbar nu** (trivial — antingen radera rad 51+113 eller
   addera `StartedAt` till record + audit-anropet, paritet `JobAdsSynced`).
   Liten polish, blockerar inte GO.

3. **Migration Down är tidsfönster-osäker — endast kommentar-skyddad**
   Fil: `src/.../Migrations/20260519064819_ResumeVersionContentEncDualShadowMapping.cs:64-74`
   `Down()` kör `ALTER COLUMN content SET NOT NULL` vilket failar 23502 så
   snart en content_enc-only-rad existerar. Detta är dokumenterat (rad 54–62)
   och är samma rollback-klass som ADR 0049 Beslut 5-precedensen — inte ett
   brott, men en operationell skarp kant. Ingen kod-fix; noteras för
   STOPP-rapporten så Klas är medveten om att rollback efter första
   content_enc-write kräver manuell data-remediering.
   §9.6: ingen åtgärd — medveten expand/contract-design, dokumenterad.

4. **`DrainAllOwnersAsync`-testhjälpare hårdkodar 1000-iterations-tak**
   Fil: `tests/JobbPilot.Worker.IntegrationTests/Security/FieldEncryptionBackfillTests.cs:614`
   Magic number `1000` i testhjälpare (kastar `InvalidOperationException`
   vid non-convergence). Test-only, inte produktionskod (§5.1 träffar inte
   testhjälpare lika hårt), men en `private const` med namn (t.ex.
   `MaxDrainIterations`) hade speglat produktions-`MaxBatchIterations`-mönstret
   och varit självdokumenterande.
   §9.6: **in-block-fixbar nu** (trivial konstant-extraktion). Kosmetiskt.

## Bra gjort

- Interceptor-par-symmetri: write/read Form A oförändrad, Form B adderad med
  exakt speglad fail-closed-/scope-differentierings-logik (autentiserad →
  kasta; system-scope → passthrough, aldrig ciphertext-exponering). CTO #3 (iv)
  konsekvent ärvd till Resume-vägen.
- `JsonSerializedVoField`-record + `JsonMap`/`Map`-separation: ren SRP/OCP —
  Form B adderad utan att röra Form A-vägen. SPOT `ContentJsonOptions`
  korrekt centraliserad + arch-test-spärrad mot regression.
- Idempotens-precision: legacy-id råfrågas PER kolumn FÖRE load (read-
  interceptorn dekrypterar i minnet → kan ej avgöra legacy efter
  materialisering). Scenario 3 bevisar byte-identisk ciphertext efter run 2 —
  stark regressionsspärr mot re-kryptering.
- `ContentLegacyJson` med `PropertySaveBehavior.Ignore` på BÅDE before/after-
  save: striktare än nödvändigt (legacy `content` kan aldrig write-back:as),
  motiverat i kommentar. Defensiv design åt rätt håll.
- Orchestrator-chassi-paritet (`HardDeleteAccountsJob`/`PurgeStaleRawPayloads`):
  per-owner try/catch, `OperationCanceledException` re-throw, alltid-audit
  (GDPR Art. 30 även 0 ägare), non-convergence-vakt, `LoggerMessage`
  source-gen. Konsekvent med husets jobb-mönster.
- C6 crypto-erasure inom ambient `BeginTransactionAsync`-try: aggregat-delete
  + audit-anonymisering + `DeleteDataKeysAsync` atomic, `catch → Rollback →
  throw`. Idempotent (0 DEK = no-op). Cross-user-test (scenario 11) bevisar
  bystander-DEK orörd.
- Testsviterna är spec-baserade, Testcontainers, fail-closed på både write
  (KMS-down ⇒ ingen rad) och read (KMS-down ⇒ kast, ingen klartext/null).
  Probe-deletion grundligt motiverad (ej tyst täckningsförlust).
- Markör-interface-konvention (`IRequiresFieldEncryptionKey`) konsekvent med
  befintliga Application-/FollowUp-/Note-usages; arch-test
  `EveryMessage_..._IsAuthenticated` spärrar markör-utan-auth-gate (§5.4).
