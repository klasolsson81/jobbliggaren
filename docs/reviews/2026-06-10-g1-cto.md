## Beslut 1 — Neutraler: **A — behåll on-disk v3-navy-neutraler, byt endast accenten**

Referens-HTML:ens slate-värden (#0F172A/#CBD5E1/#020617) är *incidentell scaffolding* från den stale design-tokens-skillen — inte ett beslut; handoff-texten säger explicit "Neutraler (oförändrade från civic-utility)". Kanon = globals.css + ADR 0052 (Accepted, användartest-grundat). Pixel-trohet mot stale underlag = cargo cult; B skulle supersea ADR 0052 Beslut 1 utan mandat (dark canvas #0B1525 "mörk navy-grå, inte svart" är ett medvetet v3-val). OCP: `--jp-brand-*`-bryggan finns exakt för denna swap (Martin 2017 kap. 8; Hunt/Thomas DRY).

**Bindande preciseringar:** (1) hero-interna kontrollvärden hero-scopas (handoffens #CBD5E1-border = banner-lokalt värde, ej global slate-neutral); (2) `--jp-hero-ink-soft` #BBCCE5 (blåtonad) ersätts av vit-alpha; (3) CC kontrast-verifierar #6EE7A8 mot faktiska v3-dark-ytor (#0B1525 ≈ 11.9:1 beräknat — AA med marginal). **Konsekvens till Klas (ej STOPP):** rendered output blir inte pixel-identisk med F4-PNG:erna i neutral-detaljer — korrekt utfall, inte avvikelse.

## Beslut 2 — Scope: (i)–(iii) bekräftade

(i) **Accent-migrationen app-wide NU** (handoff: "kan inte delas upp — halvbytt accent ser trasig ut"); hela checklistan ingår; undantag logo-kompassen. (ii) **/jobb får full F4-banner NU.** (iii) **`.jp-pagehero` + landing: färg-swap navy→grön gradient NU, full F4-komponent senare.** Skenkonflikten "övriga banners senare" (CC-PROMPT) vs "samma bannerfärg på alla sidor tills vidare" (README) löses: "senare" = F4-*kompositionen*; "samma färg" = *färgen*. Navy-band + grön accent på samma sida = två konkurrerande identitetsfärger = det halvbytta tillstånd handoffen förbjuder (CCP, Martin kap. 13).

## Beslut 3 — ADR-form: **Ny ADR 0068**, inte 0052-amendment

Supersedes (delvis) ADR 0052 Beslut 6 (navy-kontrakt→grön); Beslut 1–5 består — därför skrivs 0052 inte om. Amends ADR 0016/gradient-förbudet (scoped hero-platta-undantag). Hue-byte av hela interaktionssystemet + två principundantag som rör tre tidigare ADRs = förstklassigt beslut med egen nod (Nygard 2011 — ADR-immutabilitet, supersedes-länkning; amendment-formen är för mekanik-justeringar à la ADR 0065). **Pending docs-drift #0B5CAD→navy SUPERSEDAS:** skill-syncen går direkt till grön kanon i ett svep (YAGNI/lean — mellansteg via navy = två review-cykler för ett tillstånd som dör samma dag). CC-PROMPT kräver att undantagen dokumenteras "så design-reviewern inte revertar" — ADR 0068 + skill-text ÄR den mekanismen.

## Beslut 4 — Sekvens: E2b–E2e pausas; identiteten NU som egen PR. Fas-namn: **"G1"**

Beck: "make the change easy, then make the easy change" — varje E2-komponent byggd i blått = planerad rework. Accent-migrationen är bred men *mekanisk*; E2-arbetet *funktionellt* — blandad diff vore ogranskbar i båda dimensionerna (Fowler 2018; SWE@Google kap. 9). Inte "Fas F4" (F-numreringen upptagen av HANDOVER-v3-refaktorfaserna), inte "E-design-2" (E = paritetsarbete). "F4 Hybrid" lever som design-variantens namn inne i fasen.
