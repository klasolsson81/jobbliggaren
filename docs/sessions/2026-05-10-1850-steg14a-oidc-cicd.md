## In-flight fixar

### Fix 1 — Strict trust-policy scope (security-auditor M1 + M2)

**Problem:** Initial trust-policy hade 3 sub-claim-mönster:
```hcl
"repo:klasolsson81/jobbpilot:ref:refs/tags/v*-dev",
"repo:klasolsson81/jobbpilot:ref:refs/heads/main",
"repo:klasolsson81/jobbpilot:pull_request",
```

security-auditor M1 + M2 (båda Sec-Major): `pull_request` är för bred (matchar
fork-PRs) och `refs/heads/main` är onödig (build.yml använder inte AWS-anrop).
Båda är defense-in-depth-regression vid framtida `id-token: write`-tillägg
(privilege-escalation via accidental scope-expansion). code-reviewer flaggade
M1 också, ENIGT.

**Fix:** Ta bort båda. Strict scope:
```hcl
"repo:${var.github_owner}/${var.github_repo}:ref:refs/tags/${var.dev_tag_pattern}",
```

ENBART tag-baserad deploy-trigger. Om framtida read-only AWS-anrop i build.yml
behövs: separat `read_only`-roll med eget scope.

### Fix 2 — MTPlatform-test-runner-syntax (dotnet-architect Viktigt 2)

**Problem:** Initial syntax `dotnet test JobbPilot.sln --logger trx --results-directory TestResults`.
`--logger trx` är VSTest-syntax, fragile via kompat-shim. Native MTPlatform är
`-- --report-trx`.

**Fix iteration:** Bytte till `-- --report-trx --report-trx-filename TestResults.trx --results-directory TestResults`.
**Senare iteration:** Tog bort allt utöver `dotnet test --solution JobbPilot.sln`
helt — `--results-directory` är VSTest-only och triggar help-output i MTPlatform → exit 1.

### Fix 3 — `dotnet test --solution`-flag (CI-iter 1)

**Problem (run 25633889261):**
```
Specifying a solution for 'dotnet test' should be via '--solution'.
##[error]Process completed with exit code 1.
```

`.NET 10 SDK 10.0.201` har deprecation-meddelande på positional sln-argument
för `dotnet test` (varning som behandlas som fail). Inte för restore/build.

**Fix:** `dotnet test --solution JobbPilot.sln`

### Fix 4 — pnpm `version: 9` pin (CI-iter 1)

**Problem:** `pnpm/action-setup@v4` kräver explicit version eller
`packageManager`-fält i package.json.

**Fix iteration 1:** `version: 9` direkt i workflowen.

### Fix 5 — pnpm-workspace `packages:`-fält (CI-iter 2 + 3)

**Problem (run 25633944144):**
```
ERROR  packages field missing or empty
```

`pnpm 9+` kräver `packages:`-fält när `pnpm-workspace.yaml` finns. JobbPilot
har bara `ignoredBuiltDependencies`-config för sharp/unrs-resolver — inte ett
workspace.

**Fix iteration 2:** `pnpm install --ignore-workspace --frozen-lockfile`. Funkade
för install men inte för `pnpm lint`/`pnpm test` (de saknar workspace-flagg).

**Fix iteration 3:** Lägg till `packages: ['.']` i pnpm-workspace.yaml. Solo-
projekt deklareras explicit. Fix:ar lint/typecheck/test också.

### Fix 6 — `dotnet restore --solution` MSB1001 (CI-iter 2)

**Problem (run 25633944144):**
```
MSBUILD : error MSB1001: Unknown switch.
Switch: --solution
```

`--solution`-flagga är specifik för `dotnet test`-kommandot, inte för
`dotnet restore`/`dotnet build`. Initial fix lade till `--solution` på alla
tre vilket bröt restore + build.

**Fix iteration 2:** Backa restore + build till positional sln, behåll
`--solution` bara på test.

## Lärdomar STEG 14a

- **GitHub OIDC trust-policy-disciplin är icke-trivial.** security-auditor M1+M2
  är standard sec-fynd vid OIDC-setup — accidental scope-expansion via
  `pull_request`/`refs/heads/main`-claims när workflows inte gör AWS-anrop
  därifrån. Trust-policy ska säga *vad som är tillåtet idag*, inte *vad som
  potentiellt skulle kunna behövas*. Strict scope = rätt friction vid framtida
  workflow-edits.
- **`iam:PassRole` med både `iam:PassedToService = ecs-tasks.amazonaws.com`-
  condition + roll-allow-list är gold-standard mot privilege-escalation via
  task-def-injection.** Utan det kan en kompromisserad workflow registrera en
  task-def som refererar admin-roll → privesc.
- **`RegisterTaskDefinition` kan inte resource-scope:as via IAM** (AWS-API tar
  inga resource-arn:s — alla register-anrop kör mot `*`). Mitigation går via
  PassRole-condition snarare än Resource-restriktion.
- **`dotnet test` på sln-nivå med .NET 10 SDK 10.0.201 kräver `--solution`-flag**
  (deprecation-warning behandlas som fail). Restore + build accepterar
  positional sln, så `--solution` ska BARA appliceras på test.
- **MTPlatform-runner CLI är inte VSTest.** `--logger trx` + `--results-directory`
  är VSTest-syntax. MTPlatform har egen CLI som exponeras via `--`-separator
  (`-- --report-trx`). VSTest-kompat-shim är fragile mellan SDK-versioner.
  För CI: håll det enkelt — default stdout-output räcker, MTPlatform-flaggor
  läggs till opportunistiskt om test-failures behöver djupare analys.
- **pnpm 9+ kräver `packages:`-fält** när `pnpm-workspace.yaml` finns.
  JobbPilot har workspace.yaml för `ignoredBuiltDependencies`-config bara
  (sharp/unrs-resolver) — solo-projekt deklareras med `packages: ['.']`. Detta
  fungerade lokalt med äldre pnpm men exponeras av CI.
- **`pnpm/action-setup@v4` kräver `version`-input** om `packageManager`-fält
  saknas i package.json. Pin pnpm-version i workflow är legitim approach;
  `packageManager`-fält i package.json är cleaner long-term.
- **GitHub Actions OIDC-token livslängd vs roll-tak:** `max_session_duration`
  på rollen är *tak*, `role-duration-seconds` på workflowen är *faktisk*.
  Sätt rollen till AWS-minimum (3600=1h) men workflowen till striktare (900=15min)
  för smal blast-radius vid runner-kompromiss.
- **AWS Trust Store hanterar GitHub OIDC sedan juli 2023** — `thumbprint_list = []`
  är rekommenderad praxis. Manuell thumbprint behöver inte sättas/roteras.
- **Image-only deploy-pattern (Terraform äger task-def, CI muterar bara image-fält)**
  via `describe-task-definition → render → deploy-task-definition` håller
  IaC som single source of truth för CPU/memory/secrets/env. Race-condition
  mellan Terraform `apply` och pågående CI-deploy är teoretisk i Fas 0
  (manuella applies). Mitigation runbook-rad räcker.
- **CI-debug-cykler är dyra på iterations-tid.** 4 iterationer × ~6 min per
  push-till-result = 25 min. Lokal validate via `act` (GitHub Actions runner
  emulator) eller devcontainer-test hade kunnat fånga 3 av 4 issues före push.
  Värt att lägga till i 14b-tools.
- **`packages: ['.']`-fix i pnpm-workspace.yaml är legitim förexisterande
  config-bug** som CI exponerar. Inte en STEG 14a-regression. Att ändra
  Klas:s fil utan diskussion är OK när fixet löser uppenbar bug och bevarar
  intent (`ignoredBuiltDependencies` kvar).
