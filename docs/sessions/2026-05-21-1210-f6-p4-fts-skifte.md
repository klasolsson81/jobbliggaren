# F6 P4 FTS-skifte — PostgreSQL full-text-search-hybrid + Infrastructure-query-port

**HEAD vid session-end:** `95bcb74` (origin/main). **build-CI:** grön (run `26218140677` — backend .NET inkl. FTS-integrationstester, frontend, coverage alla gröna).

## Mål

Egen fokuserad session (ren `/clear`, startprompt genererad 2026-05-21) för att lösa det kvarstående perf-problemet från F6 P4 sök-infrastruktur-fix: ADR 0061:s GIN-trigram-index gör specifika söktermer snabba men korta vanliga svenska termer ("lärare" 18.7s) förblir långsamma. EXPLAIN ANALYZE hade bevisat fundamental trigram-selektivitet (12 980 kandidater, 7 581 falska positiva, de-TOAST-recheck av stora description-texter). CTO-dom över tre ronder: PostgreSQL FTS-hybrid, Variant (b) — en lager-refaktor som flyttar sök-kompositionen Application→Infrastructure bakom en port. Lager-refaktor förtjänade full uppmärksamhet i egen session.

## Vad som gjordes

### 1. FTS search_vector + GIN-index (commit `1b04765`)

Migration `F6P4FtsSearchVector` lägger en STORED generated column `search_vector` (`to_tsvector('swedish', title || ' ' || description)`) på `job_ads` plus ett GIN-index på kolumnen. `JobAdConfiguration` fick en shadow-property för den genererade kolumnen så EF Core känner till den utan att exponera den på domänmodellen. `TestAppDbContextFactory` fick en `IModelCustomizer`-strip — se detour-noten nedan.

### 2. Lager-refaktor bakom `IJobAdSearchQuery` (commit `1eec29c`)

Detta är sessionens kärna. FTS-LINQ-funktionerna (`websearch_to_tsquery`, `@@`, `NpgsqlTsVector`, `ts_rank`) ligger fysiskt i Npgsql-assemblyn — det finns ingen provider-agnostisk väg. Arch-testet förbjuder Npgsql-referens i Application-lagret. Lösningen: en Infrastructure-query-port `IJobAdSearchQuery` med två metoder, `SearchAsync` + `CountAsync`. Hela `JobAdSearch.ApplyCriteria`+`ApplySort`-kompositionen flyttades Application→Infrastructure bakom porten (SPOT bevarad — flyttad, inte splittrad). `JobAdSearch.cs` togs bort. De tre sök-konsumerande handlers blev tunna adaptrar. Ny `JobAdFilterCriteria`/`JobAdSearchCriteria` som SPOT delad av tre konsumenter. Ny arch-test `JobAdSearchLayerTests` låser lager-gränsen. Hybrid-query: `websearch_to_tsquery('swedish', q) OR lower(title) LIKE '%q%'`, `ts_rank` ersätter ADR 0042 Beslut D2 ILIKE-heuristik på Relevance-sort.

### 3. explain-search FTS-mode (commit `6f2769b`)

`JobbPilot.Migrate` `explain-search`-CLI-mode utökades till FTS-läget — kör EXPLAIN (ANALYZE, BUFFERS) på den nya FTS-query-planen för post-deploy-verifiering. Permanent operativt verktyg.

### 4. ADR 0062 + amendments (commit `b70b548`)

ADR 0062 NY (Accepted) — FTS-hybrid-sök + Infrastructure-query-port. ADR 0061 fick additiv amendment (trigram-trigger uppfylld, FTS-hybrid införs, trigram-indexen behålls — title aktivt i fallback-grenen, description oanvänt i q-grenen; ADR 0061 superseras EJ). ADR 0039 fick additiv amendment (`JobAdSearch`-SPOT-modulen flyttar Application→Infrastructure bakom `IJobAdSearchQuery`, SPOT-principen består). ADR 0062 Beslut 4 är ett komplementärt tredje ben till ADR 0048 join-vs-port-regeln på provider-assembly-axeln — EJ supersession. README ADR-index uppdaterat. ADR-prosa skriven av adr-keeper.

### 5. Sök-laddningsindikator (commits `57feaac` + `95bcb74`)
